实现一个不持有元素的数组

Objective-C里面的数组，放进里边的元素会被持有，会对加入的元素进行一次强引用，那么怎么才能做到加入数组的元素不被持有呢？下面介绍 NSValue (NSValueExtensionMethods)类中的方法

  • (NSValue *)valueWithNonretainedObject:(nullable id)anObject;
    @property (nullable, readonly) id nonretainedObjectValue;

第一个方法官方描述是👉This method is useful if you want to add an object to a Collection but don’t want the collection to create a strong reference to it. 翻译过来就是如果你想把一个对象加入集合类中，但是不希望这个集合类对这个对象进行强引用，那么就使用这个方法。（翻译的不是很专业，大致意思就是这样了）
下面的属性调用之后，会将Value转成Object。二者配合使用，就能实现想要的效果了。

例子

将工程设置成手动内存管理，Build Settings->搜索garbage->将Automatic Reference Counting设置成NO。

Model *model = [[Model alloc] init];
model.name = @"123";
NSLog(@"retainCount=%zd", [model retainCount]);

NSMutableArray *array = [NSMutableArray array];
[array addObject:model];
NSLog(@"retainCount=%zd", [model retainCount]);

上面两个输出的日志分别是 retainCount=1、retainCount=2，正常将一个对象加入数组中时，对象会被强引用，引用计数会被加1。

我们换一种实现方式 👇

Model *model = [[Model alloc] init];
model.name = @"123";
NSLog(@"retainCount=%zd", [model retainCount]);

NSMutableArray *array = [NSMutableArray array];
NSValue *value = [NSValue valueWithNonretainedObject:model];
[array addObject:value];
NSValue *value1 = [array firstObject];
Model *model1 = [value1 nonretainedObjectValue];
NSLog(@"retainCount=%zd", [model retainCount]);

上面两个输出的日志分别是 retainCount=1、retainCount=1，这表明，使用以上方式对object进行包装一下，就能保证数组不对Object进行强引用了。

通过这两个组合，就能实现一个不持有元素的数组了。

奇偶排序

奇偶排序的算法描述：

1，先将待排序数组中的所有奇数位与自己身后相邻的偶数位相比较，如果前者大于后者，则进行交换，直到这一趟结束。
2，然后将偶数位与自己身后相邻的奇数位相比较，如果前者大于后者，则进行交换，直到这一趟结束。
3，重复1，2的步骤，直到发现无“奇偶”，“偶奇”交换的时候，就认为排序完毕，此时退出循环。

例子

①待排序数组
1 54 100 12 5 3
②所有奇数位与身后的相邻偶数位进行交换
1 54 12 100 3 5
③所有偶数位与身后的相邻奇数位进行交换
1 12 54 3 100 5
④所有奇数位与身后的相邻偶数位进行交换
1 12 3 54 5 100
⑤所有偶数位与身后的相邻奇数位进行交换
1 3 12 5 54 100
⑥所有奇数位与身后的相邻偶数位进行交换
1 3 5 12 54 100

OC代码实现

    // 待排序数组
    NSArray *array = @[@1, @54, @100, @12, @5, @3];
    [self oddEvenSortTestWithSortArray:array];

// 排序函数
- (NSArray *)oddEvenSortTestWithSortArray:(NSArray *)array
{
    NSMutableArray *sortArray = [NSMutableArray arrayWithArray:array];
    // 是否排序完成
    BOOL isSorted = NO;
    while (!isSorted) {// 排序完成后，退出循环
        isSorted = YES;

        // 取奇数
        for (NSInteger i = 0; i < sortArray.count; i += 2) {
            if (i+1 < sortArray.count) {
                NSInteger numLeft = [sortArray[i] integerValue];
                NSInteger numRight = [sortArray[i+1] integerValue];
                if (numLeft > numRight) {
                [sortArray replaceObjectAtIndex:i withObject:@(numRight)];
                [sortArray replaceObjectAtIndex:i+1 withObject:@(numLeft)];
                isSorted = NO;
                }
            }      
        }   

        // 取偶数
        for (NSInteger i = 1; i < sortArray.count; i = i+2) {
            if (i+1 < sortArray.count) {
                NSInteger numLeft = [sortArray[i] integerValue];
                NSInteger numRight = [sortArray[i+1] integerValue];
                if (numLeft > numRight) {
                [sortArray replaceObjectAtIndex:i withObject:@(numRight)];
                [sortArray replaceObjectAtIndex:i+1 withObject:@(numLeft)];
                isSorted = NO;
                }
            }
        }
    }

    return sortArray;
}

以上就是简单的奇偶排序了

Category支持添加成员变量

在Category中使用@property也是只会生成setter和getter方法的声明，如果我们真的需要给Category增加属性的实现，需要借助于运行时的两个函数：
1）obj_setAssociatedObject
2）obj_getAssociatedObject

首先介绍一下@property，它的本质其实是ivar（实例变量）+ getter方法 + setter方法。
所以在Category中重写getter和setter方法，再添加一个ivar的变量，就可以实现@property的功能了。

举个例子，给UIView的分类添加三个属性，height（NSNumber）， page（NSInteger）， name（NSString）
👇 👇 👇

#import <UIKit/UIKit.h> //    .h文件

@interface UIView (Category)

@property (nonatomic, strong) NSNumber *height;
@property (nonatomic, assign) NSInteger page;
@property (nonatomic, copy) NSString *name;

@end

👇 👇 👇

#import "UIView+Category.h"
#import <objc/runtime.h>

@implementation UIView (Category) //    .m文件

@dynamic height, page, name;

- (void)setHeight:(NSNumber *)height
{
    objc_setAssociatedObject(self, @"height", height, OBJC_ASSOCIATION_RETAIN_NONATOMIC);
}

- (NSNumber *)height
{
    return objc_getAssociatedObject(self, @"height");
}

- (void)setName:(NSString *)name
{
    objc_setAssociatedObject(self, @"name", name, OBJC_ASSOCIATION_COPY);
}

- (NSString *)name
{
    return objc_getAssociatedObject(self, @"name");
}

- (void)setPage:(NSInteger)page
{
    objc_setAssociatedObject(self, @"page", @(page), OBJC_ASSOCIATION_RETAIN_NONATOMIC);
}

- (NSInteger)page
{
    return [(NSNumber *)objc_getAssociatedObject(self, @"page") integerValue];
}

@end

以上代码就实现了给Category动态添加属性。下面介绍这两个方法
1，objc_setAssociatedObject(id object, const void *key, id value, objc_AssociationPolicy policy)
这个函数需要四个参数：源对象，关键字，关联的对象和一个关联策略。

  • @param object： 源对象 self
  • @param key： 唯一静态变量Key
  • @param value：关联的对象
  • @param policy： 关联政策,相当于属性括号中的限定条件

typedef OBJC_ENUM(uintptr_t, objc_AssociationPolicy) {
OBJC_ASSOCIATION_ASSIGN = 0, // 👉(assign)
OBJC_ASSOCIATION_RETAIN_NONATOMIC = 1, // 👉(nonatomic, retain)
OBJC_ASSOCIATION_COPY_NONATOMIC = 3, // 👉(nonatomic, copy)
OBJC_ASSOCIATION_RETAIN = 01401, // 👉(retain)
OBJC_ASSOCIATION_COPY = 01403 // 👉(copy)
};

2，objc_getAssociatedObject(id object, const void *key)
这个函数用于获取关联对象的结果。

  • @param object： 源对象 self
  • @param key： 唯一静态变量Key

👆👆👆
利用以上的方法，就可以给Category添加属性了。
