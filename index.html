利用Autolayout实现View间隔自动调整

1、使用背景

在做需求时，常常会遇见这样的需求，一个界面中，有多个View，每个View的大小由它的内容决定。当一个View有内容时，下一个view与它之间会有一个间隔，如果没有内容的话，下一个View就会紧挨着它。如下图所示👇

图一中，四个Label的大小都是自适应的，之间的间距是10px，当第二个Label中没有文字的时候，显示如下图👇

但是当第二个Label和第三个Label同时没有文字的时候呢，显示就成了下面的样子了👇

当label中没有文字的时候，但是它们之间的间距还在，这导致UI显示就会出问题了。

2、解决方案

2.1 动态更新约束

这种方案是最直接的解决办法，也是最low的一种办法了，就是在label2，label3没有内容的时候，去调整他们与旁边视图的间距约束。使用这种方法会导致维护代码的成本太高。所以这种方法是比较初级的，很不灵活。

2.2 自定义 intrinsicContentSize 方法

- (CGSize)intrinsicContentSize

这个方法是UIView的一个方法，用来返回展示完整内容所需的最小size。比如UILabel就是根据text,attributedText等来计算他的size。当视图的内容发生变化时，可以调用这个方法让layout在下次布局时重新计算。

这种方案需要继承自View去做，对于UILabel，继承是无法解决问题的，限制性很大，也不是很好。

2.3 利用对齐矩形（alignment rect）

大家可能会直观的认为 Auto Layout 中，约束是使用 frame 来确定视图的大小和位置的，但实际上，它使用的是对称矩阵（alignment rect）这个几何元素。默认的frame和alignment rect是相等的。
UIView提供了方法，由frame得到alignment rect👇

由alignment rect得到frame👇

系统还提供了一个更简单的方法，有inset来指定frame喝alignment rect的关系👇

所以对于上边的需求来说，只要重写- (UIEdgeInsets)alignmentRectInsets这个方法，就可以实现了 👇

当然，每次继承重写这个方法，开发时很不方便，可以用 关联对象 和 method swizzled 组合起来实现一个UIView的分类👇

使用时候，哪个view需要改变可视区域大小，只要实现XHCAlignmentRectInsets这个block就可以啦。

通过这个技巧，就可以更愉快的完成产品的需求了，哈哈哈。

奇偶排序

奇偶排序的算法描述：

1，先将待排序数组中的所有奇数位与自己身后相邻的偶数位相比较，如果前者大于后者，则进行交换，直到这一趟结束。
2，然后将偶数位与自己身后相邻的奇数位相比较，如果前者大于后者，则进行交换，直到这一趟结束。
3，重复1，2的步骤，直到发现无“奇偶”，“偶奇”交换的时候，就认为排序完毕，此时退出循环。

例子

①待排序数组
1 54 100 12 5 3
②所有奇数位与身后的相邻偶数位进行交换
1 54 12 100 3 5
③所有偶数位与身后的相邻奇数位进行交换
1 12 54 3 100 5
④所有奇数位与身后的相邻偶数位进行交换
1 12 3 54 5 100
⑤所有偶数位与身后的相邻奇数位进行交换
1 3 12 5 54 100
⑥所有奇数位与身后的相邻偶数位进行交换
1 3 5 12 54 100

Category支持添加成员变量

在Category中使用@property也是只会生成setter和getter方法的声明，如果我们真的需要给Category增加属性的实现，需要借助于运行时的两个函数：
1）obj_setAssociatedObject
2）obj_getAssociatedObject

首先介绍一下@property，它的本质其实是ivar（实例变量）+ getter方法 + setter方法。
所以在Category中重写getter和setter方法，再添加一个ivar的变量，就可以实现@property的功能了。

举个例子，给UIView的分类添加三个属性，height（NSNumber）， page（NSInteger）， name（NSString）

以上代码就实现了给Category动态添加属性。下面介绍这两个方法
1，objc_setAssociatedObject(id object, const void *key, id value, objc_AssociationPolicy policy)
这个函数需要四个参数：源对象，关键字，关联的对象和一个关联策略。

  • @param object： 源对象 self
  • @param key： 唯一静态变量Key
  • @param value：关联的对象
  • @param policy： 关联政策,相当于属性括号中的限定条件

typedef OBJC_ENUM(uintptr_t, objc_AssociationPolicy) {
OBJC_ASSOCIATION_ASSIGN = 0, // 👉(assign)
OBJC_ASSOCIATION_RETAIN_NONATOMIC = 1, // 👉(nonatomic, retain)
OBJC_ASSOCIATION_COPY_NONATOMIC = 3, // 👉(nonatomic, copy)
OBJC_ASSOCIATION_RETAIN = 01401, // 👉(retain)
OBJC_ASSOCIATION_COPY = 01403 // 👉(copy)
};

2，objc_getAssociatedObject(id object, const void *key)
这个函数用于获取关联对象的结果。

  • @param object： 源对象 self
  • @param key： 唯一静态变量Key

利用以上的方法，就可以给Category添加属性了。
